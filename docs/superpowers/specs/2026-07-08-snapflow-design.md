## 1. 概述

### 1.1 一句话定义
**SnapFlow** 是一个 Flutter 移动 App。用户通过拍照 + 多图步骤，生成结构化的工作流程手册，并支持 PDF 导出。

### 1.2 目标用户
个人 / 小团队通用场景。现场作业为主，办公场景兼容。

### 1.3 核心痛点
- 现场打字困难：戴手套、强光、需要快速记录
- 流程可复用：同样的巡检 / 安装 / 排错流程反复出现
- 输出需归档：交付物常为 PDF

### 1.4 v1 范围

|---|---|---|
| 拍照 + 多图 | ✅ | |

## 2. 整体架构

### 2.1 应用名
**SnapFlow** — 拍下流程，步步留痕。

### 2.2 分层
```
┌────────────────────────────────────────┐
│           Presentation 层              │
│  Flutter Widgets / Riverpod State      │
├────────────────────────────────────────┤
│             Domain 层                  │
│  UseCases · Entities · Repositories IF │
├────────────────────────────────────────┤
│              Data 层                   │
│  Local Repo (SQLite+FS) · ASR Adapter  │
│  PDF Generator · Image Editor          │
└────────────────────────────────────────┘
```

### 2.3 关键决策
- **状态管理**：Riverpod 2
- **本地数据库**：SQLite（drift 包）
- **PDF 生成**：`pdf` + `printing` 包
- **图片编辑**：自研轻量组件（裁剪/旋转/调色） + `image` 包处理像素（马赛克/标注）
- **语音转文字**：云端 ASR API（具体厂商实现阶段定）

### 2.4 模块切分
| 模块 | 职责 | 对外接口 |
|---|---|---|
| `manual` | 手册/章节/步骤 CRUD + 排序 | `ManualRepository` |
| `capture` | 相机调用 + 多图选择 | `ImageCaptureService` |
| `editor` | 图片编辑（裁剪/调色/标注/马赛克） | `ImageEditor` |
| `voice` | 录音 + ASR 上传 + 文本返回 | `VoiceToTextService` |
| `template` | 预置模板加载 + 应用到新手册 | `TemplateService` |
| `export` | 渲染手册 → PDF 字节流 | `PdfExporter` |
| `storage` | SQLite + 文件系统统一封装 | `LocalDatabase` |

## 3. 数据模型

### 3.1 实体关系
```
Manual (手册)
  ├─ id: String (UUID)
  ├─ title: String
  ├─ coverImagePath: String?
  ├─ createdAt: DateTime
  ├─ updatedAt: DateTime
  └─ steps: List<Step>  (有序)

Step (步骤)
  ├─ id: String
  ├─ order: int
  ├─ title: String?  (允许 null — 现场可只拍图不写字)
  ├─ note: String   (非 null，可空字符串；主说明，支持语音转写结果)
  ├─ images: List<StepImage>  (有序)
  └─ optionalFields: Map<String,String>  (可填字段)

StepImage (步骤图)
  ├─ id: String
  ├─ order: int
  ├─ originalPath: String  (原图)
  ├─ editedPath: String?   (编辑后副本，可空)
  └─ thumbnailPath: String (缩略���，必备)

Template (模板 — 只读，预置 JSON)
  ├─ id, name, description
  └─ steps: List<TemplateStep>  (标题/字段预填)
```

### 3.2 存储分层
```
/data/data/com.snapflow/
├─ app.db                    ← SQLite (drift)
├─ images/
│  ├─ originals/             ← 原图 (按 manualId 分目录)
│  └─ edited/                ← 编辑后副本
└─ thumbnails/               ← 缩略图 (256px 宽)
```

### 3.3 关键设计点
1. **原图 / 编辑图 / 缩略图 三层分离**——缩略图独立存储，列表滚动流畅
2. **删除手册 → 三层目录全清**，无悬空文件
3. **`editedPath` 可空**——未编辑时 PDF 直接用原图
4. **步骤 note 字段**——主说明，语音转写结果落到这里（也可手动编辑覆盖）
5. **`optionalFields`**——给"设备编号/参数值"等可变字段留口子（不强类型化）

### 3.4 表单字段示例
```
Step(title=null, note="关闭总闸", images=[…],
     optionalFields={"电压表读数":"", "确认人":""})
```

### 3.5 排序策略
- **步骤顺序**：用 `order` int 字段，CRUD 时重排或用 gap（100, 200, 300）留插入余地
- **步骤内图片顺序**：同上
- **拖拽**：使用 Flutter `ReorderableListView`

## 4. UI 方案

### 4.1 信息架构
- **主页**：手册卡片列表（封面 + 标题 + 步数）
- **手册详情**：纵向滚动展示标题→步骤；步骤可展开/折叠
- **拍摄按钮**：常驻右下角悬浮 FAB

### 4.2 主页布局
```
┌─────────────────────────┐
│ ☰  我的手册        [+]  │
├─────────────────────────┤
│ ┌──────┐                │
│ │ 封面 │ 设备巡检SOP    │
│ │      │ 12 步 · 周一   │
│ └──────┘                │
│ ┌──────┐                │
│ │ 封面 │ 安装流程        │
│ │      │  8 步 · 上周   │
│ └──────┘                │
│                         │
│                    📷 FAB│
└─────────────────────────┘
```

**为什么是这个方案**：通用场景下 80% 时间在浏览/编辑已有手册，列表视图效率最高。模板作为新建入口的渐进增强，不前置成主页。

## 5. 关键交互流程

### 5.1 流程 A：新建手册
```
主页 [+] 按钮
   ↓
弹层：选"空白手册" or "从模板新建"
   ↓
输入标题（可空、可语音转写）
   ↓
跳转"手册详情页"，空白步骤列表
```

### 5.2 流程 B：添加步骤（核心 — 现场场景）
```
详情页 [+] 步骤
   ↓
步骤编辑页 (单步聚焦视图)
   ↓
① [📷 拍照] → 相机 → 返回图列表
   或 [🖼️ 相册] → 多选 → 加入列表
   ↓
② 图列表可：点开编辑 / 长按删除 / 拖拽排序
   ↓
③ 标题 [🎤 语音] 按住录音 → 松开 → ASR → 填入标题
   或 直接打字
   或 留空
   ↓
④ 说明 note 同样支持语音转写
   ↓
⑤ [保存] / [取消]
```

### 5.3 流程 C：图片编辑
```
步骤页点开单图 → 全屏编辑器
   ├─ 工具栏：裁剪 | 旋转 | 调色 | 标注 | 马赛克 | 完成
   ├─ 裁剪：自由比例 + 1:1 / 4:3 预设
   ├─ 调色：亮度/对比度/饱和度 三滑块
   ├─ 标注：画笔 + 箭头 + 文字气泡
   ├─ 马赛克：选区打码（敏感信息遮挡）
   └─ 完成 → 生成 editedPath，缩略图同步刷新
```

### 5.4 流程 D：语音转文字
```
按钮 [🎤] 按住 → 开始录音（带音量可视化）
   ↓
松开 → 停止 → 上传音频到 ASR 云端
   ↓
返回文本 → 填入当前光标所在字段（标题 or note）
   ↓
失败：Toast 提示 + 保留录音本地缓存（可手动重试）
```

### 5.5 流程 E：导出 PDF
```
详情页 [分享/导出] 按钮
   ↓
弹层：选导出范围（整本 / 选中步骤）
   ↓
PDF 生成进度条（异步，分块渲染避免 ANR）
   ↓
预览页（pdf 渲染组件）→ [保存到本地] / [系统分享]
```

### 5.6 流程 F：模板应用
```
新建 → 选模板 → 加载模板的步骤骨架
   ↓
弹出"模板步骤列表"预览（只读）
   ↓
确认 → 创建手册，每步 title/字段已预填
   ↓
用户进入详情页继续添加图片/微调文字
```

## 6. 错误处理 & 离线/边界

### 6.1 错误处理矩阵
| 场景 | 处理 |
|---|---|
| 相机权限被拒 | 引导去系统设置 + 提供"从相册选图"降级 |
| 相册权限被拒 | 仅用相机，仍可用 |
| ASR 网络失败 | Toast 报错 + 保留录音本地文件，用户可手动重试 |
| ASR 配额耗尽 | 提示并禁用语音入口（设置页可填 API key） |
| 磁盘空间不足 | 保存时检查；不足则阻止保存并提示清理 |
| 图片编辑失败 | 保留原图不破坏；只输出错误日志 |
| PDF 渲染失败 | 显示错误页 + 保留源数据，可重试 |
| 删除最后一步 / 唯一手册 | 二次确认 |
| 应用崩溃恢复 | 启动检查 DB 与文件系统一致性，孤儿文件清理 |

### 6.2 离线策略
- **完全可离线使用**：所有 CRUD、编辑、PDF 生成、模板都不需要网
- **仅 ASR 需联网**：失败降级为手动输入，不阻塞主流程
- **本地缓存策略**：录音文件、编辑历史 30 天后自动清理（设置项可调）

### 6.3 边界 / 极值
- 单图最大 20MB / 4096×4096（超出压缩 + 警告）
- 单手册步骤数无硬上限，按性能分页（>100 时懒加载）
- 标题字数 100、note 2000（DB 字段限制 + UI 截断提示）

### 6.4 测试策略
| 类型 | 覆盖 |
|---|---|
| **单元测试** | Domain UseCases、Repository、ASR Adapter |
| **Widget 测试** | 列表渲染、空态、拖拽交互、键盘避让 |
| **集成测试** | "拍图→编辑→保存→导出 PDF" E2E |
| **手工测试** | 真机弱网/无网、相机权限、低存储、权限被拒 |
| **覆盖率目标** | Domain/UseCases ≥ 80%，UI ≥ 50% 关键路径 |

## 7. v2 预留

以下功能不在 v1 范围，但接口与数据模型预留扩展空间：

- **长图导出**：`PdfExporter` 接口抽象 `Exporter` 基类
- **Word/Markdown 导出**：同上
- **云同步**：Repository 增加 `RemoteDataSource` 抽象
- **滤镜美颜**：`ImageEditor` 工具栏预留位
- **标签分类**：`Manual` 实体增加 `tags: List<String>` 字段（v1 不暴露 UI）
